系统设计 初学

发表于 04/25/2017 | 分类于 理论学习

系统设计主要考察一个人从宏观角度进行问题分析的能力。重点在于如何准确的了解客户的需求，并根据自己已有的知识搭建一个可行的解决方案框架。

主要步骤

  • 收集需求并分析，注意大方面到小细节，从简易到复杂，从用户期待到产品实现
  • 作出合理假设，例如假设用户特征，假设用户分布
  • 画出框架图，流程可以从前往后，或者从后往前；尽量按照一个方向走，不容易乱；
  • 识别关键点，例如瓶颈，容错等等
  • 重新设计

Scalability

  • Cache: 将大量访问的请求进行cache，使用工具包括：Memcached或者Redis，数据库的index，LinkedHashMap实现的LRU，CDN等；
  • Queue: 消息队列，用于不同进程／线程之间通信，降低偶合度，顺序处理，提高可靠性和可扩展性，使用工具包括：Kafka等；
  • Asynchronized: 异步操作，非阻塞的处理请求，接收新请求之后并不阻塞线程，而是放入另一线程中，直到完成并进行回调，框架包括：Netty；
  • Load Balancer: 负载均衡，将请求合理分配到不同机器进行处理，可使用Round Robbin或自己定义的算法，工具包括：AWS ELB；
  • Parallel: 并行计算，例如MapReduce，工作包括Hadoop， Spark等；
  • Replication: 复制数据副本，提高可靠性，但是需要考虑一致性问题，可以通过版本控制，锁，一致性检查等保证一致；
  • Sharding: 数据分片，包括Vertical和Horizontal两种，可以根据Hash值分割，可以结合replication一起使用；

除了上述主要特性之外，还需要考虑的matrix：
bandwidth，throughput，latency，failure，availability，security，SQL与NoSQL的结合等。
