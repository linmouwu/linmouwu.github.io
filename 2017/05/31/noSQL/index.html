NoSQL和MySQL的区别
ACID维基
BASE维基
CAP维基

ACID

ACID是事务（transaction）的属性，对于不支持事务的数据库搜索引擎而言，并不包含ACID属性。
原子性（Atomicity）: all or nothing; 不可能只跑一部分的transaction，必须全部执行或者全部不执行；方法包括：check point，log；
一致性（consistency）: from one valid state to another; 数据库的改变前后的状态都是一致有效的；
隔离性（isolation）: two transaction should not interfere with each other; 事务之间不相互干扰，并不知道彼此存在；方法：lock;
持久性（durability）: permanent change; 变化一旦发生就是持久化的，不会消失；

事务的隔离等级

未提交读（Read Uncommitted）：允许脏读，也就是可能读取到其他会话中未提交事务修改的数据
提交读（Read Committed）：只能读取到已经提交的数据。Oracle等多数数据库默认都是该级别 (不重复读)
可重复读（Repeated Read）：可重复读。在同一个事务内的查询都是事务开始时刻一致的，InnoDB默认级别。在SQL标准中，该隔离级别消除了不可重复读，但是还存在幻象读
串行读（Serializable）：完全串行化的读，每次读都需要获得表级共享锁，读写相互都会阻塞

脏读：读取到未commit的数据；
不重复读：一个事务内两次读取同一数据的内容不一致；
幻读：全表操作会忽视事务过程中新插入的数据，需要表级锁；

BASE

基本可用（basically available）: 系统基本可用；
软状态（soft state）: 系统有一段时间不同步；
最终一致性（eventual consistency）: 数据最终是一致的；

CAP

CAP是分布式数据库（主要是NoSQL数据库）所要满足却不能同时满足的特点。
一致性（consistency）: 所有读取都能读到最新数据；Every read receives the most recent write or an error；
可用性（availability）: 每个请求不管成功或者失败都有响应；Every request receives a (non-error) response – without guarantee that it contains the most recent write；
分区容错性（partition tolerance）: 系统正常运行，即使有节点出现一场；The system continues to operate despite an arbitrary number of messages being dropped (or delayed) by the network between nodes；

Sharding and Replication

Sharding: put different set of data into different nodes; 分片，包括水平（horizontal）和垂直（vertical）两种方式；
Replication: replicate the same data into different nodes; 复制；

  • Master-slave replication: master write, slaves read, supports reading; single node failure;
  • Peer-to-peer replication: all read and write; no single node failure;

一致性类型

  • strong
  • weak
  • eventual
  • session

一致性手段

Pessimistic（消极手段）: 预防不一致，例如用锁，实现serialization；
Optimistic（积极手段）: 允许一致但检测并回滚，例如时间戳Version Control；

Quorom: 一致性手段，pessimistic：W > N/2; R+W > N; N是节点数量；

两阶段锁2PL

实现serialize transactions。
阶段一：expanding， acquire locks；
阶段二：shrinking， release lock；

两阶段提交2PC

由一个coordinator管理是否可以提交事务。
Coordinator collect the voting from all worker; coordinator decides whether the transaction is OK or not; 保证原子性，一致性；
